# Commit Messages

## Ordnet die folgenden Commit-Messages zu. (Gut oder schlecht?). Begründe mindestens drei deiner Entscheidungen.

### 1) Gut

- Add login validation, Begründung: Sagt alles aus was geaddet wurde in das Projekt.
- Refactor Calculator for readability
- Remove unused imports
- Add null check
- Handle edge cases
- Rename variables
- Improve readability

### 2) Schlecht

- Update
- Fix bug
- stuff, Begründung: Sagt nichts aus was an dem Projekt nun geändert wurder!
- Changes in app
- final version, Begründung: Ist die final Version, aber es steht ja trotzdem nicht drinnen was man nun commited hat.
- Improve Performance
- Fix issue #23
- Refactor code
- Cleanup
- Update Calculator
- Small fixes
- Adjust logic
- Temporary fix

## Einige Commit Messages aus der ersten Spalte wirken auf den ersten Blick vielleicht „okay“, sind aber problematisch. Wähle mindestens vier Grenzfälle aus der Liste aus, und halte fest:

### final version

- Es sagt nicht viel aus was im Projekt dazu geaddet wurde.
- Es fehlt was gemacht wurde z.B. (final version of login system)
- Wenn noch hinten dran was dazu geschrieben wurde.
- 'Final Version of login system, bug fixed and tested'

### fix bug

- Welcher Bug in welcher Code wurde behoben?
- Konkrete Beschreibung des Fehlers(z.B. "NullPointerException in Calculator").
- Wenn der fehler beschrieben wurde.
- "Fix NullPointerException in Calculator.divide()"

### Add login validation

- Was wurde genau geändert
- Konkrete Beschreibung was geändert wurde
- Wenn noch dazu geschrieben was geändert wurde (z.B. Email,..)
- "Add login validation(Email)"

### Cleanup
- Was wurde aufgeräumt
- Konkrete Beschreibung, was entfernt (z.B. "Cleanup: remove debug logs from login module").
- Wenn hinten noch dazu geschrieben wird was gemacht wurde
- "Cleanup: remove debug logs from login"


## Regeln für die Projektarbeit im ITP-Unterricht definieren

- Nicht mehr als 6 Wörter (Da es sonst unübersichtlich und zu viel wird)
- English (Besser für firmen falls Leute arbeiten die andere Sprachen sprechen)
- länger als 1 Wort (Da es sonst nicht verständlich ist)
