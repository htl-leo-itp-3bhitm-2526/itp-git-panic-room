## Einige Commit Messages aus der ersten Spalte wirken auf den ersten Blick vielleicht „okay“, sind aber problematisch. Wähle mindestens vier Grenzfälle aus der Liste aus, und halte fest:

### - Cleanup
* Es ist nicht so ganz klar was aufgeräumt wurde. 
* Was vielleicht aufgeräumt wurde (File/Code)
* Wenn vielleicht noch kurz angeführt wird in welcher File bzw. welcher Code
* Ich würde schreiben "Cleanup index.html. Removed boxes".

### - Add login validation 
* Was genau geändert wurde.
* Es fehlt was genau geändert wurde. 
* Wenn vielleicht noch dazugeschrieben wird was genau geändert wurde (Passwort, Email, ...).
* "Add login validation (Password, ...)".

### - final version 
* Was wurde in der final version geändert
* Was geändert wurde
* Wenn noch dabei steht was geändert wurde.
* "final version. finished login system"

### - Rename variables
* In welcher File die Variablen umbenannt wurden.
* In welcher File die Variablen umbenannt wurden.
* Wenn noch kurz die File dabei steht
* "Rename variables. Calculator.java"
