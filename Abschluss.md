# Abschluss und Regelwerk

## Branch Strategie

* main
* features/login-logout
* added login / fixed counter logig ...

## Commit & Merge 

1. Ein Commit = eine logische Änderung
2. Commit-Nachrichten im Imperativ schreiben
3. Aussagekräftige, kurze Betreffzeile (max. 50 Zeichen)
4. Warum erklären, nicht nur was
5. Keine kaputten Builds committen
