# Commit Master

## Gut oder schlecht?

| Good    | Bad |
| :-------- | :------- |
| Add login validation | update    |
| Refactor Calculator for readability | fix bug     |
| Remove unused imports   | stuff   |
| final version  |  Changes in app |
| Fix issue #23   |  Improve performance  |
|  Update Calculator  |  Refactor code  |
| Handle edge cases   | Cleanup  |
| Rename variables   |  Small fixes |
|  Improve readability | Temporary fix  |
|  Add null check  |    |
| Adjust logic   |    |


### Bergründungen

update -> eine schlechte commitnachricht, da nicht erläutert wird was genau und wie viel geupdated wurde.

Add login validation -> gute commitnachricht da explizit die Änderung angegeben wird.

Changes in app -> schlechte commitnachricht, keine genauen Angaben nur umschrieben.

## Feinschliff

Changes in App -> Unklar was genau geändert wird, akzeptabel wenn vorher schon klar ist was genau geändert wurde, Besserung durch genaues Anführen was genau geändert wurde.

Adjust logic -> Beschrieben was gemacht wurde aber zu ungenau formuliert, akzeptabel wenn voher schon genauerers Beschreiben wurde, Besserung durch genaues angeben was geändert und gefixxt wurde.

Cleanup -> Umgangsprachlich weiß man was gemeint ist jedoch ist nicht genau beschrieben was und wie genau es durchgeführt wurde.

Improve readability -> Ebenfalls weiß man was damit gemeint ist jedoch wieder zu mager beschrieben was genau gemacht wurde.
