# Commit Master

## Gut oder schlecht?

| Good    | Bad |
| :-------- | :------- |
| Add login validation | update    |
| Refactor Calculator for readability | fix bug     |
| Remove unused imports   | stuff   |
| final version  |  Changes in app |
| Fix issue #23   |  Improve performance  |
|  Update Calculator  |  Refactor code  |
| Handle edge cases   | Cleanup  |
| Rename variables   |  Small fixes |
|  Improve readability | Temporary fix  |
|  Add null check  |    |
| Adjust logic   |    |


### Bergründungen

update -> eine schlechte commitnachricht, da nicht erläutert wird was genau und wie viel geupdated wurde.

Add login validation -> gute commitnachricht da explizit die Änderung angegeben wird.

Changes in app -> schlechte commitnachricht, keine genauen Angaben nur umschrieben.
