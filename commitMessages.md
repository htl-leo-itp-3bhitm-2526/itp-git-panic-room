# Commit Master

## Commit Messages

### Gut
- Fix issue #23 **Beschreibung: Man sieht welcher Fehler gefixt wurde.**
- Refactor code
- Add login validation 
- Remove unused imports
- Improve readability **Beschreibung: Man sieht genau was gemacht wurde.**
- Improve performance
- Refactor Calculator for readability
- Update Calculator

### Schlecht
- Cleanup
- Update **Grenzfall: Man weiß nicht was geupdated wurde, besser wäre: "Update von ...**
- stuff **Beschreibung: Man kann sich nicht denken was in diesem Commit gemacht wurde.**
- Fix bug **Grenzfall: Man weiß nicht welcher bug gefixt wurde, besser wäre: "Fix bug #..."**
- Temporary fix
- final version 
- Small fixes **Grenzfall: Man weiß nicht was gefixt wurde, besser wäre: "small fixes of ..."**
- Adjust logic **Grenzfall: Man weiß nicht welche Logik angepasst wurde, besser wäre "Adjust ... Logic"**
- Rename variables 
- Changes in app 
- Add null check
- Handle edge cases


## Regeln für Commit Messages
1. Eindeutige Commit Messages(Hust hust Clemens)
2. Muss sich um den Inhalt handeln
3. Bei wiederholenden Messages - eigenen Branch erstellen und am Schluss Mergen (Bsp. Ganz oft style von Login bearbeiten)
